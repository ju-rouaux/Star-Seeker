\doxysubsection{Description détaillée}
Include file for SDL joystick event handling

The term \char`\"{}device\+\_\+index\char`\"{} identifies currently plugged in joystick devices between 0 and \mbox{\hyperlink{SDL__joystick_8h_a2ab130eaa28f8e3bd544a5432041a311}{SDL\+\_\+\+Num\+Joysticks()}}, with the exact joystick behind a device\+\_\+index changing as joysticks are plugged and unplugged.

The term \char`\"{}instance\+\_\+id\char`\"{} is the current instantiation of a joystick device in the system, if the joystick is removed and then re-\/inserted then it will get a new instance\+\_\+id, instance\+\_\+id\textquotesingle{}s are monotonically increasing identifiers of a joystick plugged in.

The term \char`\"{}player\+\_\+index\char`\"{} is the number assigned to a player on a specific controller. For XInput controllers this returns the XInput user index. Many joysticks will not be able to supply this information.

The term Joystick\+GUID is a stable 128-\/bit identifier for a joystick device that does not change over time, it identifies class of the device (a X360 wired controller for example). This identifier is platform dependent.

In order to use these functions, \mbox{\hyperlink{SDL_8h_a8fc8d35348d7c74bad8392d776c937b8}{SDL\+\_\+\+Init()}} must have been called with the \mbox{\hyperlink{SDL_8h_ad6b47d785eaa00d9c48e6c1cff4aef4b}{SDL\+\_\+\+INIT\+\_\+\+JOYSTICK}} flag. This causes SDL to scan the system for joysticks, and load appropriate drivers.

If you would like to receive joystick updates while the application is in the background, you should set the following hint before calling \mbox{\hyperlink{SDL_8h_a8fc8d35348d7c74bad8392d776c937b8}{SDL\+\_\+\+Init()}}\+: SDL\+\_\+\+HINT\+\_\+\+JOYSTICK\+\_\+\+ALLOW\+\_\+\+BACKGROUND\+\_\+\+EVENTS

\doxysubsubsection{\texorpdfstring{SDL\_JoystickID}{SDL\_JoystickID}}
{\footnotesize\ttfamily typedef Sint32 \mbox{\hyperlink{SDL__joystick_8h_a3c3d32500cb08f76ee8077983912c0bd}{SDL\+\_\+\+Joystick\+ID}}}

This is a unique ID for a joystick for the time it is connected to the system, and is never reused for the lifetime of the application. If the joystick is disconnected and reconnected, it will get a new ID.

The ID value starts at 0 and increments from there. The value -\/1 is an invalid ID.
