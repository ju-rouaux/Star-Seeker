{\footnotesize\ttfamily DECLSPEC int SDLCALL SDL\+\_\+\+Init (\begin{DoxyParamCaption}\item[{Uint32}]{flags }\end{DoxyParamCaption})}

Initialize the SDL library.

\mbox{\hyperlink{SDL_8h_a8fc8d35348d7c74bad8392d776c937b8}{SDL\+\_\+\+Init()}} simply forwards to calling \mbox{\hyperlink{SDL_8h_adfbfddc0ec609b5e5e5cb1c89298e4db}{SDL\+\_\+\+Init\+Sub\+System()}}. Therefore, the two may be used interchangeably. Though for readability of your code \mbox{\hyperlink{SDL_8h_adfbfddc0ec609b5e5e5cb1c89298e4db}{SDL\+\_\+\+Init\+Sub\+System()}} might be preferred.

The file I/O (for example\+: SDL\+\_\+\+RWFrom\+File) and threading (SDL\+\_\+\+Create\+Thread) subsystems are initialized by default. Message boxes (SDL\+\_\+\+Show\+Simple\+Message\+Box) also attempt to work without initializing the video subsystem, in hopes of being useful in showing an error dialog when SDL\+\_\+\+Init fails. You must specifically initialize other subsystems if you use them in your application.

Logging (such as SDL\+\_\+\+Log) works without initialization, too.

{\ttfamily flags} may be any of the following OR\textquotesingle{}d together\+:


\begin{DoxyItemize}
\item {\ttfamily SDL\+\_\+\+INIT\+\_\+\+TIMER}\+: timer subsystem
\item {\ttfamily SDL\+\_\+\+INIT\+\_\+\+AUDIO}\+: audio subsystem
\item {\ttfamily SDL\+\_\+\+INIT\+\_\+\+VIDEO}\+: video subsystem; automatically initializes the events subsystem
\item {\ttfamily SDL\+\_\+\+INIT\+\_\+\+JOYSTICK}\+: joystick subsystem; automatically initializes the events subsystem
\item {\ttfamily SDL\+\_\+\+INIT\+\_\+\+HAPTIC}\+: haptic (force feedback) subsystem
\item {\ttfamily SDL\+\_\+\+INIT\+\_\+\+GAMECONTROLLER}\+: controller subsystem; automatically initializes the joystick subsystem
\item {\ttfamily SDL\+\_\+\+INIT\+\_\+\+EVENTS}\+: events subsystem
\item {\ttfamily SDL\+\_\+\+INIT\+\_\+\+EVERYTHING}\+: all of the above subsystems
\item {\ttfamily SDL\+\_\+\+INIT\+\_\+\+NOPARACHUTE}\+: compatibility; this flag is ignored
\end{DoxyItemize}

Subsystem initialization is ref-\/counted, you must call \mbox{\hyperlink{SDL_8h_a14e9091982e09097985e10e3ca65debb}{SDL\+\_\+\+Quit\+Sub\+System()}} for each \mbox{\hyperlink{SDL_8h_adfbfddc0ec609b5e5e5cb1c89298e4db}{SDL\+\_\+\+Init\+Sub\+System()}} to correctly shutdown a subsystem manually (or call \mbox{\hyperlink{SDL_8h_afdabaf714781099083592051f2d9ac11}{SDL\+\_\+\+Quit()}} to force shutdown). If a subsystem is already loaded then this call will increase the ref-\/count and return.

0 on success or a negative error code on failure; call \mbox{\hyperlink{SDL__error_8h_ac88e1aa404a8f94aaee4ad6b94547647}{SDL\+\_\+\+Get\+Error()}} for more information.

{\footnotesize\ttfamily DECLSPEC void SDLCALL SDL\+\_\+\+Quit (\begin{DoxyParamCaption}\item[{void}]{ }\end{DoxyParamCaption})}

Clean up all initialized subsystems.

You should call this function even if you have already shutdown each initialized subsystem with \mbox{\hyperlink{SDL_8h_a14e9091982e09097985e10e3ca65debb}{SDL\+\_\+\+Quit\+Sub\+System()}}. It is safe to call this function even in the case of errors in initialization.

If you start a subsystem using a call to that subsystem\textquotesingle{}s init function (for example \mbox{\hyperlink{SDL__video_8h_a05244b30716f0339dcd140c6f92017b7}{SDL\+\_\+\+Video\+Init()}}) instead of \mbox{\hyperlink{SDL_8h_a8fc8d35348d7c74bad8392d776c937b8}{SDL\+\_\+\+Init()}} or \mbox{\hyperlink{SDL_8h_adfbfddc0ec609b5e5e5cb1c89298e4db}{SDL\+\_\+\+Init\+Sub\+System()}}, then you must use that subsystem\textquotesingle{}s quit function (\mbox{\hyperlink{SDL__video_8h_ade5fc4847320fb235f17265b683fa013}{SDL\+\_\+\+Video\+Quit()}}) to shut it down before calling \mbox{\hyperlink{SDL_8h_afdabaf714781099083592051f2d9ac11}{SDL\+\_\+\+Quit()}}. But generally, you should not be using those functions directly anyhow; use \mbox{\hyperlink{SDL_8h_a8fc8d35348d7c74bad8392d776c937b8}{SDL\+\_\+\+Init()}} instead.

You can use this function with atexit() to ensure that it is run when your application is shutdown, but it is not wise to do this from a library or other dynamically loaded code.

Shut down specific SDL subsystems.

If you start a subsystem using a call to that subsystem\textquotesingle{}s init function (for example \mbox{\hyperlink{SDL__video_8h_a05244b30716f0339dcd140c6f92017b7}{SDL\+\_\+\+Video\+Init()}}) instead of \mbox{\hyperlink{SDL_8h_a8fc8d35348d7c74bad8392d776c937b8}{SDL\+\_\+\+Init()}} or \mbox{\hyperlink{SDL_8h_adfbfddc0ec609b5e5e5cb1c89298e4db}{SDL\+\_\+\+Init\+Sub\+System()}}, \mbox{\hyperlink{SDL_8h_a14e9091982e09097985e10e3ca65debb}{SDL\+\_\+\+Quit\+Sub\+System()}} and \mbox{\hyperlink{SDL_8h_a04dc570990c697eed95681216801710d}{SDL\+\_\+\+Was\+Init()}} will not work. You will need to use that subsystem\textquotesingle{}s quit function (\mbox{\hyperlink{SDL__video_8h_ade5fc4847320fb235f17265b683fa013}{SDL\+\_\+\+Video\+Quit()}}) directly instead. But generally, you should not be using those functions directly anyhow; use \mbox{\hyperlink{SDL_8h_a8fc8d35348d7c74bad8392d776c937b8}{SDL\+\_\+\+Init()}} instead.

You still need to call \mbox{\hyperlink{SDL_8h_afdabaf714781099083592051f2d9ac11}{SDL\+\_\+\+Quit()}} even if you close all open subsystems with \mbox{\hyperlink{SDL_8h_a14e9091982e09097985e10e3ca65debb}{SDL\+\_\+\+Quit\+Sub\+System()}}.
