{\footnotesize\ttfamily DECLSPEC \mbox{\hyperlink{SDL__video_8h_a55a196c7d3b8497538632c79ae1e6392}{SDL\+\_\+\+Window}}$\ast$ SDLCALL SDL\+\_\+\+Create\+Window (\begin{DoxyParamCaption}\item[{const char $\ast$}]{title,  }\item[{int}]{x,  }\item[{int}]{y,  }\item[{int}]{w,  }\item[{int}]{h,  }\item[{Uint32}]{flags }\end{DoxyParamCaption})}

Create a window with the specified position, dimensions, and flags.

{\ttfamily flags} may be any of the following OR\textquotesingle{}d together\+:


\begin{DoxyItemize}
\item {\ttfamily SDL\+\_\+\+WINDOW\+\_\+\+FULLSCREEN}\+: fullscreen window
\item {\ttfamily SDL\+\_\+\+WINDOW\+\_\+\+FULLSCREEN\+\_\+\+DESKTOP}\+: fullscreen window at desktop resolution
\item {\ttfamily SDL\+\_\+\+WINDOW\+\_\+\+OPENGL}\+: window usable with an Open\+GL context
\item {\ttfamily SDL\+\_\+\+WINDOW\+\_\+\+VULKAN}\+: window usable with a Vulkan instance
\item {\ttfamily SDL\+\_\+\+WINDOW\+\_\+\+METAL}\+: window usable with a Metal instance
\item {\ttfamily SDL\+\_\+\+WINDOW\+\_\+\+HIDDEN}\+: window is not visible
\item {\ttfamily SDL\+\_\+\+WINDOW\+\_\+\+BORDERLESS}\+: no window decoration
\item {\ttfamily SDL\+\_\+\+WINDOW\+\_\+\+RESIZABLE}\+: window can be resized
\item {\ttfamily SDL\+\_\+\+WINDOW\+\_\+\+MINIMIZED}\+: window is minimized
\item {\ttfamily SDL\+\_\+\+WINDOW\+\_\+\+MAXIMIZED}\+: window is maximized
\item {\ttfamily SDL\+\_\+\+WINDOW\+\_\+\+INPUT\+\_\+\+GRABBED}\+: window has grabbed input focus
\item {\ttfamily SDL\+\_\+\+WINDOW\+\_\+\+ALLOW\+\_\+\+HIGHDPI}\+: window should be created in high-\/\+DPI mode if supported ($>$= SDL 2.\+0.\+1)
\end{DoxyItemize}

{\ttfamily SDL\+\_\+\+WINDOW\+\_\+\+SHOWN} is ignored by \mbox{\hyperlink{SDL__video_8h_a42565088bdb739a5353fd67abbe235d2}{SDL\+\_\+\+Create\+Window()}}. The SDL\+\_\+\+Window is implicitly shown if SDL\+\_\+\+WINDOW\+\_\+\+HIDDEN is not set. {\ttfamily SDL\+\_\+\+WINDOW\+\_\+\+SHOWN} may be queried later using \mbox{\hyperlink{SDL__video_8h_a4445733f04a1a9dd0e063cd4738f285f}{SDL\+\_\+\+Get\+Window\+Flags()}}.

On Apple\textquotesingle{}s mac\+OS, you {\bfseries{must}} set the NSHigh\+Resolution\+Capable Info.\+plist property to YES, otherwise you will not receive a High-\/\+DPI Open\+GL canvas.

If the window is created with the {\ttfamily SDL\+\_\+\+WINDOW\+\_\+\+ALLOW\+\_\+\+HIGHDPI} flag, its size in pixels may differ from its size in screen coordinates on platforms with high-\/\+DPI support (e.\+g. i\+OS and mac\+OS). Use \mbox{\hyperlink{SDL__video_8h_a5cbfffcfec91c22a7ca95fd1d5163db5}{SDL\+\_\+\+Get\+Window\+Size()}} to query the client area\textquotesingle{}s size in screen coordinates, and \mbox{\hyperlink{SDL__video_8h_ac21851bbd91760c5bab92594a58edba3}{SDL\+\_\+\+GL\+\_\+\+Get\+Drawable\+Size()}} or \mbox{\hyperlink{SDL__render_8h_abfc0c9a50d9d1870ab7d271b7a73d8ab}{SDL\+\_\+\+Get\+Renderer\+Output\+Size()}} to query the drawable size in pixels.

If the window is set fullscreen, the width and height parameters {\ttfamily w} and {\ttfamily h} will not be used. However, invalid size parameters (e.\+g. too large) may still fail. Window size is actually limited to 16384 x 16384 for all platforms at window creation.

If the window is created with any of the SDL\+\_\+\+WINDOW\+\_\+\+OPENGL or SDL\+\_\+\+WINDOW\+\_\+\+VULKAN flags, then the corresponding Load\+Library function (SDL\+\_\+\+GL\+\_\+\+Load\+Library or SDL\+\_\+\+Vulkan\+\_\+\+Load\+Library) is called and the corresponding Unload\+Library function is called by \mbox{\hyperlink{SDL__video_8h_a72609295a4faee0587268339ccff80c2}{SDL\+\_\+\+Destroy\+Window()}}.

If SDL\+\_\+\+WINDOW\+\_\+\+VULKAN is specified and there isn\textquotesingle{}t a working Vulkan driver, \mbox{\hyperlink{SDL__video_8h_a42565088bdb739a5353fd67abbe235d2}{SDL\+\_\+\+Create\+Window()}} will fail because \mbox{\hyperlink{SDL__vulkan_8h_a9fcc0942c8eaa67c3eff2e3526304b5b}{SDL\+\_\+\+Vulkan\+\_\+\+Load\+Library()}} will fail.

If SDL\+\_\+\+WINDOW\+\_\+\+METAL is specified on an OS that does not support Metal, \mbox{\hyperlink{SDL__video_8h_a42565088bdb739a5353fd67abbe235d2}{SDL\+\_\+\+Create\+Window()}} will fail.

On non-\/\+Apple devices, SDL requires you to either not link to the Vulkan loader or link to a dynamic library version. This limitation may be removed in a future version of SDL.

{\footnotesize\ttfamily DECLSPEC void$\ast$ SDLCALL SDL\+\_\+\+GL\+\_\+\+Get\+Proc\+Address (\begin{DoxyParamCaption}\item[{const char $\ast$}]{proc }\end{DoxyParamCaption})}

Get an Open\+GL function by name.

If the GL library is loaded at runtime with \mbox{\hyperlink{SDL__video_8h_a398df9aaf43a69dc2cfaab5b306b8f91}{SDL\+\_\+\+GL\+\_\+\+Load\+Library()}}, then all GL functions must be retrieved this way. Usually this is used to retrieve function pointers to Open\+GL extensions.

There are some quirks to looking up Open\+GL functions that require some extra care from the application. If you code carefully, you can handle these quirks without any platform-\/specific code, though\+:


\begin{DoxyItemize}
\item On Windows, function pointers are specific to the current GL context; this means you need to have created a GL context and made it current before calling \mbox{\hyperlink{SDL__video_8h_a6329bc32811639fe6f71606801cd9e7b}{SDL\+\_\+\+GL\+\_\+\+Get\+Proc\+Address()}}. If you recreate your context or create a second context, you should assume that any existing function pointers aren\textquotesingle{}t valid to use with it. This is (currently) a Windows-\/specific limitation, and in practice lots of drivers don\textquotesingle{}t suffer this limitation, but it is still the way the wgl API is documented to work and you should expect crashes if you don\textquotesingle{}t respect it. Store a copy of the function pointers that comes and goes with context lifespan.
\item On X11, function pointers returned by this function are valid for any context, and can even be looked up before a context is created at all. This means that, for at least some common Open\+GL implementations, if you look up a function that doesn\textquotesingle{}t exist, you\textquotesingle{}ll get a non-\/\+NULL result that is {\itshape NOT} safe to call. You must always make sure the function is actually available for a given GL context before calling it, by checking for the existence of the appropriate extension with \mbox{\hyperlink{SDL__video_8h_a1fa0cc176fbc16db139b40a121e9cfa7}{SDL\+\_\+\+GL\+\_\+\+Extension\+Supported()}}, or verifying that the version of Open\+GL you\textquotesingle{}re using offers the function as core functionality.
\item Some Open\+GL drivers, on all platforms, {\itshape will} return NULL if a function isn\textquotesingle{}t supported, but you can\textquotesingle{}t count on this behavior. Check for extensions you use, and if you get a NULL anyway, act as if that extension wasn\textquotesingle{}t available. This is probably a bug in the driver, but you can code defensively for this scenario anyhow.
\item Just because you\textquotesingle{}re on Linux/\+Unix, don\textquotesingle{}t assume you\textquotesingle{}ll be using X11. Next-\/gen display servers are waiting to replace it, and may or may not make the same promises about function pointers.
\item Open\+GL function pointers must be declared {\ttfamily APIENTRY} as in the example code. This will ensure the proper calling convention is followed on platforms where this matters (Win32) thereby avoiding stack corruption.
\end{DoxyItemize}

Some systems allow specifying -\/1 for the interval, to enable adaptive vsync. Adaptive vsync works the same as vsync, but if you\textquotesingle{}ve already missed the vertical retrace for a given frame, it swaps buffers immediately, which might be less jarring for the user during occasional framerate drops. Provide a callback that decides if a window region has special properties.

Normally windows are dragged and resized by decorations provided by the system window manager (a title bar, borders, etc), but for some apps, it makes sense to drag them from somewhere else inside the window itself; for example, one might have a borderless window that wants to be draggable from any part, or simulate its own title bar, etc.

This function lets the app provide a callback that designates pieces of a given window as special. This callback is run during event processing if we need to tell the OS to treat a region of the window specially; the use of this callback is known as \char`\"{}hit testing.\char`\"{}

Mouse input may not be delivered to your application if it is within a special area; the OS will often apply that input to moving the window or resizing the window and not deliver it to the application.

Specifying NULL for a callback disables hit-\/testing. Hit-\/testing is disabled by default.

Platforms that don\textquotesingle{}t support this functionality will return -\/1 unconditionally, even if you\textquotesingle{}re attempting to disable hit-\/testing.

Your callback may fire at any time, and its firing does not indicate any specific behavior (for example, on Windows, this certainly might fire when the OS is deciding whether to drag your window, but it fires for lots of other reasons, too, some unrelated to anything you probably care about {\itshape and when the mouse isn\textquotesingle{}t actually at the location it is testing}). Since this can fire at any time, you should try to keep your callback efficient, devoid of allocations, etc.

Initialize the video subsystem, optionally specifying a video driver.

This function initializes the video subsystem, setting up a connection to the window manager, etc, and determines the available display modes and pixel formats, but does not initialize a window or graphics mode.

If you use this function and you haven\textquotesingle{}t used the SDL\+\_\+\+INIT\+\_\+\+VIDEO flag with either \mbox{\hyperlink{SDL_8h_a8fc8d35348d7c74bad8392d776c937b8}{SDL\+\_\+\+Init()}} or \mbox{\hyperlink{SDL_8h_adfbfddc0ec609b5e5e5cb1c89298e4db}{SDL\+\_\+\+Init\+Sub\+System()}}, you should call \mbox{\hyperlink{SDL__video_8h_ade5fc4847320fb235f17265b683fa013}{SDL\+\_\+\+Video\+Quit()}} before calling \mbox{\hyperlink{SDL_8h_afdabaf714781099083592051f2d9ac11}{SDL\+\_\+\+Quit()}}.

It is safe to call this function multiple times. \mbox{\hyperlink{SDL__video_8h_a05244b30716f0339dcd140c6f92017b7}{SDL\+\_\+\+Video\+Init()}} will call \mbox{\hyperlink{SDL__video_8h_ade5fc4847320fb235f17265b683fa013}{SDL\+\_\+\+Video\+Quit()}} itself if the video subsystem has already been initialized.

You can use \mbox{\hyperlink{SDL__video_8h_ac6f3250a95bd027fd1a374d7ed15fc57}{SDL\+\_\+\+Get\+Num\+Video\+Drivers()}} and \mbox{\hyperlink{SDL__video_8h_a2ae1ace02d55dbed8ab2fc35dadac139}{SDL\+\_\+\+Get\+Video\+Driver()}} to find a specific {\ttfamily driver\+\_\+name}.
