Wait until a condition variable is signaled.

This function unlocks the specified {\ttfamily mutex} and waits for another thread to call \mbox{\hyperlink{SDL__mutex_8h_a1037a8274b0b13118fc78eeb417a1397}{SDL\+\_\+\+Cond\+Signal()}} or \mbox{\hyperlink{SDL__mutex_8h_a2ffd0b45ebdc44057007a59857ba46b3}{SDL\+\_\+\+Cond\+Broadcast()}} on the condition variable {\ttfamily cond}. Once the condition variable is signaled, the mutex is re-\/locked and the function returns.

The mutex must be locked before calling this function.

This function is the equivalent of calling \mbox{\hyperlink{SDL__mutex_8h_aa57013f95a9c608cf24a5100b841e863}{SDL\+\_\+\+Cond\+Wait\+Timeout()}} with a time length of {\ttfamily SDL\+\_\+\+MUTEX\+\_\+\+MAXWAIT}.


\begin{DoxyParams}{Paramètres}
{\em cond} & the condition variable to wait on \\
\hline
{\em mutex} & the mutex used to coordinate thread access \\
\hline
\end{DoxyParams}
\begin{DoxyReturn}{Renvoie}
0 when it is signaled or a negative error code on failure; call \mbox{\hyperlink{SDL__error_8h_ac88e1aa404a8f94aaee4ad6b94547647}{SDL\+\_\+\+Get\+Error()}} for more information.

Create a new mutex.

All newly-\/created mutexes begin in the {\itshape unlocked} state.

Calls to \mbox{\hyperlink{SDL__mutex_8h_a0908c6a7356331d197059fbdc6c66a0f}{SDL\+\_\+\+Lock\+Mutex()}} will not return while the mutex is locked by another thread. See \mbox{\hyperlink{SDL__mutex_8h_a5097fb4c7147d14a4c6df4515ec862e8}{SDL\+\_\+\+Try\+Lock\+Mutex()}} to attempt to lock without blocking.

SDL mutexes are reentrant.

{\footnotesize\ttfamily DECLSPEC SDL\+\_\+sem$\ast$ SDLCALL SDL\+\_\+\+Create\+Semaphore (\begin{DoxyParamCaption}\item[{Uint32}]{initial\+\_\+value }\end{DoxyParamCaption})}

Create a semaphore.

This function creates a new semaphore and initializes it with the value {\ttfamily initial\+\_\+value}. Each wait operation on the semaphore will atomically decrement the semaphore value and potentially block if the semaphore value is 0. Each post operation will atomically increment the semaphore value and wake waiting threads and allow them to retry the wait operation.

Lock the mutex.

This will block until the mutex is available, which is to say it is in the unlocked state and the OS has chosen the caller as the next thread to lock it. Of all threads waiting to lock the mutex, only one may do so at a time.
